¿Qué es la Regresión Lineal?

Es el "Hola Mundo" del Machine Learning, pero sigue siendo el caballo de batalla de la industria.

El Objetivo

Encontrar la relación entre:

  • Variable Independiente (\(X\)): El dato que tenemos (Input).
  • Variable Dependiente (\(Y\)): El dato que queremos predecir (Target/Output).

Requisito Fundamental

La variable a predecir (\(Y\)) debe ser continua (números reales como precio, temperatura, ventas).

Si \(Y\) fuera categórica (Sí/No), usaríamos Clasificación, no Regresión.

3

La Ecuación de la Recta

Todo el modelo se resume en esta fórmula matemática simple:

$$Y = \beta_0 + \beta_1 X + \epsilon$$
  • \(Y\): La predicción final (variable dependiente).
  • \(X\): El dato de entrada (variable independiente).
  • \(\epsilon\): El error aleatorio (lo que el modelo no puede explicar).
  • \(\beta_0\) (Intercepto): ¿Cuánto vale \(Y\) si \(X = 0\)? (Punto de partida).
  • \(\beta_1\) (Pendiente): ¿Cuánto cambia \(Y\) si \(X\) sube 1 unidad? (La fuerza de la relación).
4

Visualizando la Linealidad

Imagina que queremos predecir el Precio de una casa según su Tamaño.

El modelo busca dibujar la línea roja que pase "más cerca" de todos los puntos negros.

5

Mínimos Cuadrados Ordinarios (OLS)

¿Cómo decide el ordenador cuál es la "mejor" línea entre infinitas posibilidades?

La Estrategia

El algoritmo calcula la distancia vertical (error o residuo) entre cada punto real y la línea propuesta.

Luego eleva esas distancias al cuadrado y las suma. Se elevan al cuadrado para penalizar errores grandes y evitar que errores positivos y negativos se cancelen.

Objetivo OLS: Minimizar la Suma de los Errores al Cuadrado: $$\min_{\beta_0, \beta_1} \sum_{i=1}^{n} \epsilon_i^2 = \sum_{i=1}^{n} (y_i - \hat{y}_i)^2$$
6

Flujo de Trabajo en Python

Usaremos scikit-learn, el estándar de la industria.

# 1. Importaciones
import pandas as pd
from sklearn.linear_model import LinearRegression

# 2. Preparar datos
# X debe ser siempre 2D (Tabla/DataFrame), y debe ser 1D (Serie)
df = pd.DataFrame({
    'superficie': [50, 60, 80, 100],
    'precio':     [150, 180, 220, 280]
})

X = df[['superficie']]  # Doble corchete → DataFrame (2D)  ✓
y = df['precio']        # Un corchete    → Series (1D)     ✓
7

Entrenando el Modelo

El método .fit() es donde ocurre la magia matemática (el cálculo de OLS).

# 3. Inicializar el modelo
modelo = LinearRegression()

# 4. Entrenar: ajustar la recta a los datos
modelo.fit(X, y)

print("¡Modelo entrenado exitosamente!")
En este punto, el objeto modelo ya ha calculado internamente los valores óptimos para \(\beta_0\) (intercepto) y \(\beta_1\) (pendiente) que minimizan la suma de errores al cuadrado.
8

Interpretación de Resultados

No basta con ejecutar el código, hay que leer lo que nos dice.

print(f"Intercepto (β₀): {modelo.intercept_:.1f}")  # → 10.5
print(f"Pendiente  (β₁): {modelo.coef_[0]:.1f}")    # → 2.5

Traducción al Negocio:

  • Intercepto \(\hat{\beta}_0 = 10.5\): "El precio base estimado —incluso con 0 m² de superficie— es 10 500 €. (Interpretación teórica; fuera del rango de los datos puede carecer de sentido práctico.)"
  • Pendiente \(\hat{\beta}_1 = 2.5\): "Por cada metro cuadrado adicional que tenga la casa, el precio sube 2 500 € en promedio, manteniendo el resto de factores constante."
9

Predicciones (Inferencia)

Una vez entrenado, usamos el modelo para estimar el valor de casos desconocidos.

# Queremos saber el precio para casas de 130 y 150 m²
nuevos_datos = pd.DataFrame({'superficie': [130, 150]})

predicciones = modelo.predict(nuevos_datos)

print(predicciones)
# Salida esperada: [335.5  385.5]

El modelo aplica la fórmula \(\hat{Y} = 10.5 + 2.5 \cdot X\) automáticamente:

  • \(\hat{Y}(130) = 10.5 + 2.5 \times 130 = 10.5 + 325 = \mathbf{335.5}\)
  • \(\hat{Y}(150) = 10.5 + 2.5 \times 150 = 10.5 + 375 = \mathbf{385.5}\)

Regresión Lineal Múltiple

El mundo real es complejo. El precio de una casa no depende solo del tamaño.

Regresión Simple

1 Variable Predictora (\(X\))

Precio ~ Tamaño

Regresión Múltiple

Múltiples Variables Predictoras (\(X_1, X_2, \ldots\))

Precio ~ Tamaño + Habitaciones + Antigüedad + Barrio

11

La Ecuación se Expande

Simplemente añadimos más términos a nuestra suma.

$$Y = \beta_0 + \beta_1 X_1 + \beta_2 X_2 + \cdots + \beta_p X_p + \epsilon$$

Cada variable (\(X_j\)) tiene su propio coeficiente o "peso" (\(\beta_j\)) que indica cuánto influye en el resultado final, manteniendo las demás variables constantes.

Nota sobre notación: Usamos \(p\) para el número de predictores. El modelo tiene en total \(p + 1\) parámetros: \(\beta_0, \beta_1, \ldots, \beta_p\).
12

Interpretación "Ceteris Paribus"

Una frase en latín crucial para los economistas y data scientists: "Si todo lo demás permanece constante".

Ejemplo de Interpretación en Regresión Múltiple

Supongamos el modelo: \(\text{Precio} = \beta_0 + \beta_1 \cdot \text{Tamaño} + \beta_2 \cdot \text{Habitaciones} + \beta_3 \cdot \text{Antigüedad}\)

Si \(\hat{\beta}_2 = 5\,000\):

"Añadir 1 habitación más se asocia con un aumento de 5 000 € en el precio, asumiendo que el tamaño y la antigüedad de la casa NO CAMBIAN."

Esto aísla el efecto de una variable específica, controlando el ruido de las demás. Es la gran ventaja de la regresión múltiple sobre analizar cada variable por separado.
13

Ejemplo Real: California Housing

Vamos a predecir precios de casas usando múltiples factores del dataset California Housing de Scikit-Learn.

import pandas as pd
from sklearn.datasets import fetch_california_housing
from sklearn.linear_model import LinearRegression

data = fetch_california_housing()
X = pd.DataFrame(data.data, columns=data.feature_names)
y = data.target  # Precio medio en cientos de miles de dólares

# Seleccionamos 3 variables representativas
X_subset = X[['MedInc', 'HouseAge', 'AveRooms']]

modelo_multi = LinearRegression()
modelo_multi.fit(X_subset, y)

print("Intercepto:", modelo_multi.intercept_)
print("Coeficientes:", dict(zip(X_subset.columns, modelo_multi.coef_)))
14

Leyendo los Resultados Reales

La ecuación ajustada es: \(\hat{Y} = \beta_0 + 0.45 \cdot \text{MedInc} + 0.01 \cdot \text{HouseAge} - 0.05 \cdot \text{AveRooms}\)

Variable Coeficiente (\(\hat{\beta}\)) Interpretación (ceteris paribus)
MedInc (Ingreso Medio) \(+0.45\) A mayor ingreso medio del barrio, mayor precio de la vivienda. (Relación lógica y esperada.)
HouseAge (Edad de la Casa) \(+0.01\) Casas más antiguas valen ligeramente más. Posible explicación: ubicaciones céntricas o factor "vintage".
AveRooms (Promedio de Cuartos) \(-0.05\) Coeficiente negativo: más habitaciones por bloque puede indicar particiones pequeñas o zonas rurales de menor valor.
Los signos de los coeficientes no siempre son intuitivos. Siempre interpretar en el contexto de las demás variables incluidas en el modelo.
15

Peligro: Multicolinealidad

¿Qué pasa si incluimos dos variables que dicen lo mismo? (Ej: "Tamaño en pies" y "Tamaño en metros")

El Problema

El modelo se confunde. No sabe a cuál variable atribuirle el mérito sobre la predicción.

Resultado: coeficientes inestables, con signos inesperados o con alta varianza.

Cómo Detectarla

Calcula la matriz de correlación entre tus variables \(X\):

import seaborn as sns
corr = X_subset.corr()
sns.heatmap(corr, annot=True, cmap='coolwarm')
Regla práctica:

Si dos variables tienen una correlación \(|r| > 0.9\), elimina una de ellas antes de entrenar.

Para análisis más rigurosos, usa el Factor de Inflación de la Varianza (VIF): un VIF > 10 indica multicolinealidad problemática.
16

Diagnóstico: Análisis de Residuos

En regresión múltiple no podemos dibujar el hiperplano en 2D. Diagnosticamos el modelo a través de sus residuos \(e_i = y_i - \hat{y}_i\).

preds    = modelo_multi.predict(X_subset)
residuos = y - preds

import matplotlib.pyplot as plt
plt.scatter(preds, residuos, alpha=0.4, color='black')
plt.axhline(0, color='red', linewidth=1.5)
plt.xlabel("Valores Predichos (Ŷ)")
plt.ylabel("Residuos (e = Y - Ŷ)")
plt.title("Gráfico de Residuos")
plt.show()
Supuesto clave: Los residuos deben ser independientes, tener media cero y varianza constante (homocedasticidad).

Si los puntos forman una nube aleatoria alrededor del cero → modelo correcto.
Si aparece una curva o embudo → puede haber no-linealidad o heterocedasticidad.

Resumen Ejecutivo

La Base

Ajustamos una recta (o hiperplano) minimizando la suma de errores cuadráticos (OLS): \(\min \sum \epsilon_i^2\).

La Herramienta

LinearRegression() de Scikit-Learn. .fit(X, y) para entrenar, .predict(X_new) para inferir.

Las Precauciones

Vigila la multicolinealidad (\(|r| > 0.9\)). Valida el modelo con el gráfico de residuos. Interpreta coeficientes siempre en contexto.
