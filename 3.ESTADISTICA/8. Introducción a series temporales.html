¿Qué es una Serie Temporal?

Es una secuencia de observaciones \(y_1, y_2, \ldots, y_T\) donde el orden temporal es fundamental: lo que pasó ayer influye en lo que ocurre hoy.

Datos Transversales

El orden de las filas no importa. Podemos barajar las observaciones sin perder información.

Ej: Encuesta de satisfacción a 100 clientes. Da igual encuestar primero al cliente 1 o al 50.

Serie Temporal

El orden define la información. Barajar los datos destruye el significado.

Ej: Ventas diarias, precio de una acción, temperatura horaria. El pasado condiciona el futuro.

Consecuencia técnica: En series temporales no podemos usar Train/Test Split aleatorio. El test set debe ser siempre el bloque de tiempo más reciente, para que el modelo nunca "vea el futuro".
3

Los Componentes Base

Cualquier serie temporal puede descomponerse matemáticamente en una "receta" de cuatro ingredientes. El modelo más común es el aditivo:

$$Y_t = T_t + S_t + C_t + \varepsilon_t$$

\(T_t\) — Tendencia

Dirección general a largo plazo. Crecimiento o declive sostenido.

\(S_t\) — Estacionalidad

Patrón periódico y predecible (diario, semanal, anual…).

\(C_t\) — Ciclo

Fluctuaciones de largo plazo sin periodo fijo (ciclos económicos).

\(\varepsilon_t\) — Ruido

Componente aleatoria irreducible. Lo que ningún modelo puede capturar.

Cuando la amplitud de la estacionalidad crece con la tendencia, se usa el modelo multiplicativo: \(Y_t = T_t \times S_t \times C_t \times \varepsilon_t\).
4

1. Tendencia (Trend)

La dirección general a largo plazo: la "pendiente" subyacente que persiste ignorando las fluctuaciones temporales.

  • ¿Crece el número de usuarios de la plataforma?
  • ¿Bajan los costes de producción año tras año?
  • Puede ser lineal, exponencial o incluso decreciente, pero es consistente en el tiempo.
La tendencia se extrae matemáticamente promediando (media móvil de ventana grande) o ajustando una regresión sobre el tiempo \(t\).
5

2. Estacionalidad (Seasonality)

Patrones que se repiten con periodo fijo y predecible. A diferencia de la tendencia, la estacionalidad vuelve siempre al mismo nivel base.

  • Anual: Ventas de helados (pico en verano).
  • Semanal: Tráfico web (cae el fin de semana).
  • Diaria: Consumo eléctrico (sube por la noche).
  • Intradiaria: Llamadas a call center (pico a mediodía).
La estacionalidad tiene un periodo \(s\) conocido. Detectarlo es clave: \(s=7\) para datos diarios con patrón semanal, \(s=12\) para datos mensuales con patrón anual.
6

3. Ruido / Residuo (\(\varepsilon_t\))

Lo aleatorio e impredecible. Es la parte de la serie que ningún modelo puede —ni debe— capturar.

Un buen modelo debería aislar la Tendencia y la Estacionalidad para que solo quede Ruido. Si los residuos todavía muestran un patrón, el modelo está dejando escapar información útil.

¿Cómo sabemos si es ruido puro?

El ruido ideal tiene media cero, varianza constante y sin autocorrelación (los valores no se influyen entre sí). Esto se verifica con el ACF Plot.

Preparando Pandas para Series Temporales

El índice del DataFrame debe ser de tipo DatetimeIndex. Sin esto, las operaciones temporales (resample, rolling, shift…) no funcionarán correctamente.

import pandas as pd

# 1. Cargar el CSV
df = pd.read_csv('ventas.csv')

# 2. Convertir la columna de fecha a tipo datetime
df['fecha'] = pd.to_datetime(df['fecha'])

# 3. Establecer la fecha como índice
df.set_index('fecha', inplace=True)

# Verificar
print(df.index.dtype)      # → datetime64[ns]
print(df.index.freq)       # → None (frecuencia irregular) o 'D' (diario), etc.

# Si la frecuencia no está definida, establecerla explícitamente:
df = df.asfreq('D')        # 'D'=diario, 'M'=mensual, 'H'=horario

Resampling (Cambio de Frecuencia)

A veces los datos diarios tienen demasiado ruido y necesitamos ver el panorama general agrupando a una frecuencia menor.

# Agrupar por Mes y calcular la Media
df_mensual = df['ventas'].resample('M').mean()

# Agrupar por Semana y Sumar las ventas
df_semanal = df['ventas'].resample('W').sum()

# Agrupar por Trimestre
df_trimestral = df['ventas'].resample('Q').sum()

# Códigos frecuentes:
# 'D' → Diario   'W' → Semanal
# 'M' → Mensual  'Q' → Trimestral
# 'Y' → Anual    'H' → Horario

¿Por qué hacerlo?

El resampling suaviza la curva y resalta patrones de largo plazo que están ocultos en el ruido de alta frecuencia.

Upsampling vs Downsampling: Pasar de diario a mensual es downsampling (menos puntos). Pasar de mensual a diario es upsampling y requiere interpolación para rellenar los valores faltantes.
9

Suavizado: Medias Móviles (Rolling Mean)

Técnica para extraer la tendencia calculando el promedio de una ventana deslizante de \(n\) periodos anteriores.

$$\text{MA}_t(n) = \frac{1}{n} \sum_{k=0}^{n-1} y_{t-k} = \frac{y_t + y_{t-1} + \cdots + y_{t-n+1}}{n}$$

Efecto de la ventana \(n\)

\(n\) pequeño (ej. 7 días): sigue de cerca los datos, suaviza poco.

\(n\) grande (ej. 90 días): curva muy suave, resalta solo la tendencia estructural.

Coste de la ventana: Los primeros \(n-1\) valores del resultado serán NaN, porque no hay suficiente historia. Usa .rolling(n, min_periods=1) si necesitas evitarlo.
10

Implementación: Rolling Windows

Pandas hace que calcular y visualizar medias móviles sea sencillo con el método .rolling().

import matplotlib.pyplot as plt

# Media Móvil de 7 días (tendencia semanal)
df['ma_7']  = df['ventas'].rolling(window=7).mean()

# Media Móvil de 30 días (tendencia mensual)
df['ma_30'] = df['ventas'].rolling(window=30).mean()

# Graficar las tres series
fig, ax = plt.subplots(figsize=(12, 5))
ax.plot(df['ventas'],  alpha=0.35, color='gray',   label='Serie original')
ax.plot(df['ma_7'],   color='orange', linewidth=1.5, label='MA(7) — semanal')
ax.plot(df['ma_30'],  color='red',    linewidth=2,   label='MA(30) — mensual')
ax.legend()
ax.set_title('Ventas diarias con medias móviles')
plt.tight_layout()
plt.show()

Descomposición Automática con statsmodels

La función seasonal_decompose separa matemáticamente los componentes en un solo paso.

from statsmodels.tsa.seasonal import seasonal_decompose
import matplotlib.pyplot as plt

# model='additive'      → Y_t = T_t + S_t + ε_t
# model='multiplicative' → Y_t = T_t × S_t × ε_t
# period: nº de observaciones por ciclo (365 para datos diarios con patrón anual)

result = seasonal_decompose(df['ventas'], model='additive', period=365)
fig = result.plot()
fig.set_size_inches(12, 8)
plt.tight_layout()
plt.show()

Output: 4 subgráficos apilados

La función genera automáticamente: (1) Serie original, (2) Tendencia \(T_t\), (3) Estacionalidad \(S_t\) y (4) Residuo \(\varepsilon_t\). Si el residuo parece ruido aleatorio, la descomposición fue exitosa.

Concepto Clave: Estacionariedad

La mayoría de modelos de predicción (ARIMA, etc.) asumen que las propiedades estadísticas de la serie no cambian con el tiempo.

✓ Serie Estacionaria

✗ Serie No Estacionaria

Es Estacionaria si…

✅ Media constante en el tiempo (\(\mathbb{E}[Y_t] = \mu\)).

✅ Varianza constante (\(\text{Var}(Y_t) = \sigma^2\)).

✅ La autocorrelación solo depende del lag \(k\), no del tiempo \(t\).

No es Estacionaria si…

❌ Tiene tendencia (media que sube o baja).

❌ Tiene estacionalidad marcada sin eliminar.

❌ La varianza crece con el tiempo (heterocedasticidad).

13

Validación Estadística: ADF Test

No adivines si la serie es estacionaria. El Augmented Dickey-Fuller Test lo contrasta formalmente.

from statsmodels.tsa.stattools import adfuller

result  = adfuller(df['ventas'])
p_value = result[1]
stat    = result[0]

print(f"Estadístico ADF: {stat:.4f}")
print(f"P-Value:         {p_value:.4f}")
print(f"Valores críticos: {result[4]}")
Hipótesis del test:
H₀ (nula): La serie tiene raíz unitaria → No es estacionaria.
H₁ (alternativa): La serie es estacionaria.

Regla de decisión: Si el p-value < 0.05 → rechazamos H₀ → la serie es estacionaria.
Si el p-value > 0.05 → no podemos rechazar H₀ → debemos transformar la serie (diferenciación, log…).
14

Solución: Diferenciación (Differencing)

Cuando la serie no es estacionaria por tener tendencia, la solución más habitual es calcular la primera diferencia: el cambio de un periodo al siguiente.

$$Y'_t = Y_t - Y_{t-1} = \Delta Y_t$$ 
# Primera diferencia
df['diff_1'] = df['ventas'].diff()

# Si una diferencia no basta (tendencia cuadrática):
df['diff_2'] = df['ventas'].diff().diff()

# Verificar con ADF después de diferenciar
from statsmodels.tsa.stattools import adfuller
p = adfuller(df['diff_1'].dropna())[1]
print(f"P-Value tras diff: {p:.4f}")  # Debe ser < 0.05

¿Por qué funciona?

Si la tendencia es lineal (\(Y_t = a + bt + \varepsilon_t\)), al diferenciar obtenemos \(Y'_t = b + \varepsilon_t - \varepsilon_{t-1}\), que ya tiene media constante.

El número de diferencias necesarias se denomina orden de integración \(d\). Es el parámetro \(d\) del modelo ARIMA(\(p, d, q\)).
15

Autocorrelación (ACF)

La Función de Autocorrelación mide cuánto se parece la serie a una versión desplazada de sí misma (\(k\) pasos atrás, llamados lags).

from statsmodels.graphics.tsaplots import plot_acf, plot_pacf
import matplotlib.pyplot as plt

fig, (ax1, ax2) = plt.subplots(2, 1, figsize=(12, 6))

# ACF: Autocorrelación con todos los lags previos (directa + indirecta)
plot_acf(df['ventas'], lags=50, ax=ax1)
ax1.set_title('ACF — Función de Autocorrelación')

# PACF: Autocorrelación parcial (solo el efecto directo del lag k)
plot_pacf(df['ventas'], lags=50, ax=ax2)
ax2.set_title('PACF — Autocorrelación Parcial')

plt.tight_layout()
plt.show()
Interpretación: Las barras que superan la banda azul (intervalo de confianza al 95%) indican lags estadísticamente significativos. Picos rítmicos cada 7 lags en datos diarios confirman estacionalidad semanal.
16

Lag Plots: Visualizando la "Memoria"

Un scatter plot donde el eje X es \(Y_{t-1}\) (valor de ayer) y el eje Y es \(Y_t\) (valor de hoy). Muestra visualmente si el pasado predice el presente.

from pandas.plotting import lag_plot
import matplotlib.pyplot as plt

fig, axes = plt.subplots(1, 2, figsize=(10, 4))

# Lag 1: correlación con ayer
lag_plot(df['ventas'], lag=1, ax=axes[0])
axes[0].set_title('Lag 1 (ayer vs hoy)')

# Lag 7: correlación con el mismo día de la semana pasada
lag_plot(df['ventas'], lag=7, ax=axes[1])
axes[1].set_title('Lag 7 (misma semana)')

plt.tight_layout()
plt.show()

Diagonal fuerte (↗)

Alta autocorrelación. Los valores de ayer predicen bien los de hoy. La serie tiene "memoria".

Nube de puntos (⬛)

Sin autocorrelación. La serie es ruido aleatorio (random walk si la causa es la tendencia). Ayer no dice nada de hoy.
